## Objective
Make security testing a recurring process, not a one-time effort.

## Tasks
- Add PR template checklist for security impact + test evidence.
- Schedule quarterly OWASP regression assessments.
- Define exception process (`docs/security/exceptions.md`) with owner and expiry.
- Track KPIs: coverage %, open high/critical counts, MTTR.
- Plan annual external penetration test.

## Acceptance Criteria
- PR checklist live and enforced by reviewers.
- Recurring calendar/issue automation in place.
- KPI dashboard/report published monthly.
- Security exceptions reviewed at least monthly.

## Deliverables
- `docs/security/exceptions.md`
- `docs/security/security-kpis.md`
